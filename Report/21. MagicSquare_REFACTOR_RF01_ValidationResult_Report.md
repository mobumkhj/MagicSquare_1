## 4. 구현 상세 (C1)

### 4.1 변경 파일

| 파일 | 변경 유형 | 내용 |
|------|-----------|------|
| `src/boundary/validation_result.py` | **신규** | `@dataclass ValidationResult` — `ok()` / `fail(code, message)` |
| `src/boundary/input_validator.py` | 수정 | `_validate_core()` → `ValidationResult`; `validate()` → envelope 매핑 |
| `src/boundary/schemas.py` | 수정 | `FailureResponse` 클래스·`dataclass` import **제거** |

### 4.2 아키텍처 (ECB)

```
InputValidator.validate(grid)
  └─▶ _validate_core(grid) → ValidationResult   [검증 판정 SSOT]
        └─▶ _to_error_response(result) → ErrorResponse | None   [envelope]
              └─▶ PuzzleBoundary.submit() → control (변경 없음)
```

| 레이어 | C1 영향 |
|--------|---------|
| Boundary | ValidationResult VO · InputValidator SRP 개선 |
| Control | **변경 없음** |
| Entity | **변경 없음** |
| Screen | **변경 없음** |

### 4.3 핵심 코드 구조

**ValidationResult** (`validation_result.py`):

- `is_valid: bool`, `code: str | None`, `message: str | None`
- `ValidationResult.ok()` / `ValidationResult.fail(code, message)`

**InputValidator** (`input_validator.py`):

- public `validate()` 시그니처·반환 타입 **유지** — AC-FR-01-01 테스트가 `.code`/`.message` 직접 접근
- 검증 규칙은 `_validate_core()`에 집중; envelope 생성은 `_to_error_response()` 단일 경로

## 2. Criterion 1 — 모든 FR에 테스트 가능한 AC가 있는가?

### FR-01 Input Verification — 부분 충족

**있는 AC:** 4종 오류 코드, Domain 미호출.

| 문제 | 영향 | 개선안 |
|------|------|--------|
| `message` 바이트 동일 AC 없음 | 오류 문구 회귀 미방지 | AC: message가 §13 문자열과 완전 일치 |
| 성공 경로 AC 없음 | 검증 통과 후 흐름 미검증 | AC: 유효 입력 → solver 1회 호출 |
| jagged/3×4 등 세부 AC 없음 | INVALID_SIZE 커버리지 불완전 | AC: 3×4, jagged → INVALID_SIZE + 미호출 |

### FR-02 Blank Coordinate Discovery — 미흡

| 문제 | 영향 | 개선안 |
|------|------|--------|
| AC 2줄 수준(동일 입력, firstBlank) | 자동화 시 기대값 불명확 | Given matrix → Then firstBlank/secondBlank 1-index 고정값 |
| secondBlank AC 없음 | row-major 둘째 빈칸 회귀 | AC: secondBlank 좌표 검증 |
| Input/Output/Error/Test Direction 필드 누락 | FR 템플릿 불완전 | 템플릿 필드 보완 |

### FR-03 Missing Number Discovery — 미충족

| 문제 | 영향 | 개선안 |
|------|------|--------|
| AC: “누락 2개, small < large”만 | 측정 불가에 가까움 | Given 행렬 → Then (small,large) 구체값 |
| Processing Rules 등 필드 누락 | Domain 계약 불명확 | FR-01 수준 필드 보완 |

### FR-04 Magic Square Validation — 미흡

| 문제 | 영향 | 개선안 |
|------|------|--------|
| AC가 I1~I3 참조, PRD에 I1~I3 미정의 | 추적·테스트 기준 모호 | BR-08~11과 1:1 매핑 또는 Invariant 표 추가 |
| 부정 케이스 AC 없음 | 판정 로직 회귀 | rowOk_diagFail, sum34_wrongSet 등 분리 AC |

### FR-05 Two-Combination Solver — 미흡

| 문제 | 영향 | 개선안 |
|------|------|--------|
| AC 요약 수준 | Attempt별 n 순서·좌표-빈칸 대응 미검증 | (r1,c1)=firstBlank, Attempt1: n1<n2, Attempt2: n1>n2 |
| Layer Domain+Boundary 혼합 | Dual-Track AC 분리 불가 | FR-05a Domain / FR-05b Boundary 분리 |
| 양쪽 실패 AC 없음 | DOMAIN_NO_SOLUTION 미보호 | AC: both fail → DOMAIN_NO_SOLUTION |

## 3. Criterion 2 — 모든 AC가 Traceability Matrix에 연결되어 있는가?

**판정: 미충족**

### 구조적 문제

- §21 Matrix에 **Acceptance Criteria 열 없음** (작성 지시 §21 형식과 불일치).
- Concept → Test Case만 있고 FR-XX AC 문장과 1:1 연결 불가.

### Matrix에 없는 AC/요구

| 누락 항목 | 관련 FR/섹션 |
|-----------|--------------|
| Domain solver 미호출 | FR-01 AC |
| 오류 message 고정 | FR-01, §13 |
| 성공 시 code 필드 없음 | Report/02 U-12 |
| DOMAIN_NO_SOLUTION / 양쪽 실패 | FR-05, §13 |
| 결정성 | NFR-03 |
| 입력 0-index / 출력 1-index | Report/02, §12 |
| (r1,c1)=firstBlank 대응 | FR-05 |
| NFR (커버리지, 50ms, no side effects) | §14 |
| Track A 미호출 관찰 | §15.1 |

**개선안:** Matrix에 AC ID 열 추가, 각 FR AC 최소 1행 매핑, NFR Traceability 확장.

## 4. Criterion 3 — Boundary와 Domain 책임이 섞인 곳은 없는가?

**판정: 부분 충족**

| 위치 | 문제 | 개선안 |
|------|------|--------|
| FR-05 Layer `Domain + Boundary` | Solver·포맷·반환 한 FR | FR-05a/b 분리 |
| FR-05 Component `PuzzleBoundary` | Boundary가 solver FR에 포함 | Track B에서 Boundary 제외 |
| ResultFormatter `Domain 또는 Control` | 출력 계약 책임 미확정 | Control 1종 확정 권장 |
| FR-02 Output 1-index 내부 | Domain vs Boundary 포맷 경계 | Formatter Layer 확정 |
| Control AC 없음 | 오케스트레이션 미검증 | FR-06 Control 추가 |

## 5. Criterion 4 — 오류 정책이 미정으로 남아 있지 않은가?

**판정: 미충족** (작성 지시 “미정 금지”와 충돌)

### 확정된 부분

- Boundary 4종 + `DOMAIN_NO_SOLUTION` code/message (§13).
- 입력 검증 실패 시 Domain 미호출.

### 미정/누락

| 항목 | 위치 | 개선안 |
|------|------|--------|
| Domain 내부 실패 표현 | §13 Decision Needed | Result/Failure vs 예외 1종 확정 |
| NoValidCompletion 변환 | §13 | 호출 주체·타입 명시 |
| Domain 방어 실패 | FR-02~03 | INTERNAL_ERROR 또는 “발생하지 않음” 고정 |
| INTERNAL_ERROR | §13 없음 | Report/02 반영 |
| 성공 응답 schema | §12 | success에 code 없음 명시 |

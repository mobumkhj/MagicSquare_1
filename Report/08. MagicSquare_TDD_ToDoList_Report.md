# MagicSquare 4x4 개발 To-Do List

## 1. Epic

- [ ] Epic-001: 불변식 기반 사고 훈련 시스템 구축 — 4×4 Magic Square 입력/출력 계약 고정, Dual-Track UI + Logic TDD, ECB(boundary → control → entity) 분리, Concept-to-Code Traceability 유지

---

## 2. User Stories

- [ ] US-001: 입력 검증(Boundary) — 4×4·빈칸 2개·값 범위·중복 규칙 위반 시 표준 오류를 반환하고 Domain Solver를 호출하지 않는다
- [ ] US-002: 빈칸 좌표 탐색(Entity) — row-major 순서로 첫 번째·두 번째 빈칸(0) 좌표를 1-index로 결정한다
- [ ] US-003: 누락 숫자 탐색(Entity) — 1~16 중 입력에 없는 숫자 2개를 오름차순(small, large)으로 반환한다
- [ ] US-004: 마방진 검증(Entity) — 완성된 4×4 격자의 행·열·대각선 합이 Magic Constant 34인지 판정한다
- [ ] US-005: 두 조합 Solver(Control + Entity) — small-first → reverse 순으로 시도하고, 성공 시 `[r1,c1,n1,r2,c2,n2]`를 반환하며 양쪽 실패 시 도메인 실패를 반환한다
- [ ] US-006: Boundary 통합 및 출력 계약(Boundary + Control) — 유효 입력에 대해 `int[6]` 성공 응답 또는 `DOMAIN_NO_SOLUTION` 오류를 표준 형식으로 반환한다

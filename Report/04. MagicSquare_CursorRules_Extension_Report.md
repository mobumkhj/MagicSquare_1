## 1. 목적

기존 루트의 단일 규칙 파일 `.cursorrules`에 작성된 규칙을 **주제별로 분리**하여, Cursor가 상황(열려 있는 파일 종류/경로)에 맞춰 더 정확하게 규칙을 적용할 수 있도록 `.cursor/rules/*.mdc` 형태로 확장한다.

## 2. 변경 요약

### 2.1 입력(기존)

- 단일 파일: `.cursorrules`
- 포함 섹션: `project`, `code_style`, `architecture`, `tdd_rules`, `testing`, `forbidden`, `file_structure`, `ai_behavior`

### 2.2 출력(분리 결과)

`.cursor/rules/` 아래에 총 **7개** 규칙 파일로 분리했다.

| 파일 | 적용 범위 | 요약 |
|------|----------|------|

## 3. 분류 기준(매핑)

`.cursorrules` 섹션을 다음 기준으로 나눴다.

- **프로젝트 공통/운영**: 목표, 제약, “절대 하면 안 되는 진행 방식” → `magicsquare-project.mdc`
- **언어/스타일**: PEP8, 타입힌트, docstring 등 → `magicsquare-python-code-style.mdc` (Python 파일에서만 적용)
- **아키텍처**: ECB 역할/의존성 방향/입출력 위치 → `magicsquare-ecb-architecture.mdc`
- **테스트/TDD**: pytest/AAA, RED/GREEN/REFACTOR, 커버리지 → `magicsquare-tdd-testing.mdc` (테스트 파일에서만 적용)
- **금지 규칙**: 출력 디버깅, 매직 넘버, broad except 등 → `magicsquare-forbidden.mdc`
- **구조 가이드**: 디렉터리 배치 권장 → `magicsquare-file-structure.mdc`
- **AI 작업 규칙**: 코딩 전/중/후 체크 → `magicsquare-ai-behavior.mdc`

---

## 4. 규칙 적용 방식(의도)

- “항상 적용” 규칙은 프로젝트 전반의 일관성을 위해 `alwaysApply: true`로 설정했다.
- 파일별 규칙은 다음처럼 **특정 파일에서만 적용**되도록 `globs`를 사용했다.
  - Python 스타일: `**/*.py`
  - 테스트/TDD 상세: `**/tests/**/*.py`

## 2. 배경 및 설계 근거

### 2.1 Report/02 Screen Layer 정합

| Report/02 항목 | 본 구현 |
|----------------|---------|
| Caller → `PuzzleBoundary.submit(matrix)` | `PuzzlePresenter.present(grid)` → `PuzzleBoundary.solve(grid)` |
| InputValidator 선행 검증 | 동일 — invalid 시 Domain 미호출 |
| 성공 → `int[6]` | `format_success()` 로 GUI 표시 |
| 실패 → `ErrorResponse` (code, message) | `format_error()` 로 GUI 표시 |
| Domain 실패 → `DOMAIN_NO_SOLUTION` | `PuzzleBoundary.submit()` 에서 `UnsolvableDomainError` catch |

### 2.2 ECB 의존성

```
MainWindow (screen I/O)
  → PuzzlePresenter (screen orchestration)
    → PuzzleBoundary (boundary)
      → InputValidator / SolveTwoBlanksUseCase (control)
        → entity services
```

| 규칙 | 준수 |
|------|:----:|
| I/O는 boundary(screen)에서만 | ✅ |
| screen → boundary (entity/control 직접 호출 금지) | ✅ |
| boundary → control → entity | ✅ |
| Domain 예외 매핑은 boundary (`PuzzleBoundary`) | ✅ |

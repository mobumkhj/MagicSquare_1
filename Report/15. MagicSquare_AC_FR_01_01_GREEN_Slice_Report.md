## 2. GREEN 슬라이스 상세

### 2.1 Slice 1 — Domain 격리 (`grid=None`)

| 항목 | 값 |
|------|-----|
| **테스트** | `tests/boundary/test_ac_fr_01_01_isolation.py::TestAcFr0101IsolationVerification::test_none_grid_submit_resolve_execute_not_called` |
| **RED 원인** | `ModuleNotFoundError: No module named 'src.control'` (fixture setup) |
| **GREEN 결과** | **PASSED** — `mock_resolve_use_case.execute.assert_not_called()` |
| **AC 계약** | `type="ERROR"`, `code="INVALID_SIZE"`, `message="Grid must be 4x4."` |

**구현 요약**

- `src/boundary/schemas.py` — `FailureResponse` dataclass, 계약 상수
- `src/boundary/input_validator.py` — `grid is None` 선행 분기
- `src/boundary/puzzle_boundary.py` — 검증 실패 시 `execute` 미호출 early return
- `src/control/solve_two_blanks_use_case.py` — mock `spec` 충족용 클래스·`execute` 스텁 (`NotImplementedError`)

> **범위 노트:** 격리 테스트는 `PuzzleBoundary.submit()` 및 Control import를 요구하므로, Slice 1 GREEN에는 허용 목록(3 boundary 파일) 외 최소 스텁이 포함되었다.

### 2.2 Slice 2 — 경계값 빈 리스트 (`grid=[]`)

| 항목 | 값 |
|------|-----|
| **테스트** | `tests/boundary/test_ac_fr_01_01_boundary_values.py::TestAcFr0101BoundaryValues::test_empty_list_grid_returns_invalid_size_code` |
| **RED 원인** | `InputValidator.validate([])` → `NotImplementedError` |
| **GREEN 결과** | **PASSED** — `result.code == INVALID_SIZE_CODE` |
| **수정 파일** | `src/boundary/input_validator.py` (1파일) |

**구현 요약**

```python
if grid is None or grid == []:
    return FailureResponse(
        type=RESPONSE_TYPE_ERROR,
        code=INVALID_SIZE_CODE,
        message=INVALID_SIZE_MESSAGE,
    )
```

- 3×4·4×3·5×5·`[[]]*4` 등 **일반 size 검증은 미구현** (의도적 GREEN 범위 제한)
- `grid == []`만 추가; 전체 `_check_size()` 추출은 REFACTOR 단계로 미룸
